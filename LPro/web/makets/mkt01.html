Указанная ширина для контейнера суммируется с указанным отступом к содержимому, в итоге числовое значение свойства width не будет определять истинную ширину, а только слагаемую часть. Поэтому отступ padding получилось сделать без жертв только центральному, у которого значение ширины отсутствует. Слева по задумке находится меню (nav). Картинка масштабировалась языком html, просто получила значение ширины 100%, что стянуло её в рамки. Справа произвольный раздел для каких-нибудь целей (sdb). Универсальный селектор * стёр все поля, затем они были переопределены. Границы дбавлены для наглядности, если их убрать и добавить полей сверху и с боков - можно добиться миловидности. В теле три раздела просто сделуют по порядку <div/div>. Очевидный момент этой системы - неоходимость корректировать значение боковых полей центра (cnt). Из-за выпадения из потока он сам ровно не встанет, поэтому задавать поля по бокам нужно к ширинам столбцов. Также смущает перекрытие столбцов при сужении экрана, но сужение всегда ломает идею, поэтому не нужно учитывать это, нужно искать иные решения. Вопрос, который требует решения - постоянное меню. Если оно динамично, придётся редактировать код всех документов.
