Глава 1. Общий вид документа по изучаемой книге.

Все элементы кода заключены в скобки, именуемые тегами, <> вот его. Некоторые должны закрываться этим </>.

Якобы важное значение у объявления, деклараций, вступительного оформления. Хотя смысл скорее информационный, нежели механически значимый.

<!DOCTYPE html>

Вот это в заголовке кода сей страницы. Разъяснение - скобка и ! означает объявление, доктип означает, что объявляется тип документа. Хтмл строчными буквани означает объявление использованного кода, нужен для проверки чем-то чего-то. Паблик означает открытость, стало быть, что-то могло означать закрытость, но это узнаем потом. Далее идёт в кавычках узел с типом и версией языка. Затем полный адрес местоположения документа с определением.

Ниже корневой элемент, в нём полагается быть всему документу, имеющему рабочие функции.

<html>
</html>

Заголовочный элемент, включает заголовочные описания. Располагается в начале корневого элемента.

<head>
</head>

В заголовке расположены элементы следующего сожержания:
<title>Название вкладки браузера, обязательный атрибут<title/>
<meta>Кодировка, имя, ключи, автор, ряд других функций<meta/>
<script>Скрипты<script/>
<style>Стили, в данном случае встроенный способ<style/>
<link>Связка со внешними таблицами стилей, значком, и прочим<link/>

Уже есть часть информации о мета-элементе. Вот более подробно. Их можно использовать несколько с разными значениями атрибутов.

<meta charset="UTF-8" />
<meta name="description" content="описание проекта" />
<meta name="keywords" content="ключевые слова, вероятно для поиска" />
<meta name="author" content="автор" />

Корневой элемент сожержит не только голову, но и тело. По аналогии с некоторыми языками программирования, далее упоминается элемент для включения в себя всего тела сайта. Однако мой браузер выполняет даже то, что появляется вне его, и даже не входит в тело. Текст, использованный без разметки, тоже отображается свободно. Поэтому здесь код делит смысл как на механику разметки, так и на банальную этику процесса кодировки.

<body>Вот тут расположено тело, для всех отображаемых составляющих.
</body>

Итак, шаблон для моего проекта:

<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8" />
<meta name="description" content="обучающий проект, главная" />
<meta name="keywords" content="ключ, обучение, хтмл" />
<meta name="author" content="Хей Лонг" />
</head>
<body>
</body>
</html>

Этика создания документа предполагает использование элементов для структурирования информации. Для декорации или доводки расположения есть специальные способы. Заголовки, абзацы, перенос строки, комментарии, ссылки, прочие элементы оформления должны представлять собой грамотную конструкцию, а не паутину, которая рассыпется прахом при малейшей доработке.

<p>
Здесь размещается абзац. Браузер оформляет его автоматически, с переносом красной строки после закрывающего тега. Многие закрывающие теги тоже переносят строку, а ещё это можно сделать "вручную", используя <br>
и получается
вот так.
</p>

<strong>текст будет жирным</strong>, раньше был болд, но с хтмл5 вроде пропал.
<em>текст будет курсивом</em>.
<small>текст будет мелким</small>.
<!-- Этот текст не будет, он виден только в коде. -->.

<br> переносит строку вниз, хотя не отодвигает ниже к следующему абзацу.

<hr> рисует горизонтальную линию. Её видно выше, между главами.

<ol> ordered list, создаёт упорядоченный список
<li>list item, элемент списка 1</li>
<li>list item, элемент списка 2</li>
<li>list item, элемент списка 3</li>
</ol>

    ordered list, создаёт упорядоченный список
  1. list item, элемент списка 1
  2. list item, элемент списка 2
  3. list item, элемент списка 3

<ul> unordered list, создаёт неупорядоченный список
<li>list item, элемент списка 1</li>
<li>list item, элемент списка 2</li>
<li>list item, элемент списка 3</li>
</ul>

    unordered list, создаёт неупорядоченный список
  • list item, элемент списка 1
  • list item, элемент списка 2
  • list item, элемент списка 3

Разрешается использовать вложение списков. Весь список помещается между тегами элемента списка.

Глава 3. Мультимедиа. Скрипты.

Вставить картинку можно так
<img src="адрес" width="X" height="Y" alt="текст до/вместо/всплывающий картины" border="Z">

Х - ширина. Использованный % будет относиться к ширине окна браузера, и в таком случае Y будет необязателен, % к высоте не применяется. Y - высота изображения. Эти параметры определяют сжатие документа, обрезка не производится браузером.

Ссылки на файлы. Эффект не всегда предсказуемый. Загрузка в кэш браузера для воспроизведения, или предложение записи на жёсткий диск. Проверим. Вес у них минимальный.
