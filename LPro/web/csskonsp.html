Глава 1. Общая идея, использование, приоритет.

    Типы таблиц стилей.
  • Стиль браузера. Используется стандартно, если не был переопределён.
  • Пользовательский. Тот же, что и браузерный, но модифицирован пользователем.
  • Встроенный. Используется непосредственно в элементе, селекторе.
  • Внедрённый. В головном разделе документа, в элементе style формируется таблица.
  • Связанный. Таблица, написанная внешним файлом, и соединённая через link головного раздела
  • Импортированный. Как связанный, только может быть назначен в другую таблицу или в документ

Приоритет таков. Пользовательский стиль переопределяет все иные. Встроенный действует приоритетнее внедрённого, связанного и импортного. Внедрённый должен иметь преимущество перед связанным и импортным, а эти равны по переопределению. Стиль браузера будет определять только всё остальное, не имеющее определения стилем.

Наследование. Если определить стиль для элемента body, почти все элементы унаследуют совместимые свойства. Хотя это встроенный стиль, по эффекту похож на внедрённый.

Пример встроенного стиля, прямо в жирный шрифт добавим свойства размера и цвета.

<b style="color:red; font-size:20px">Пример =3</b>

Пример =3

Пример внедрённого стиля. Пишется в заголовке документа.

<style type="text/css">
body {background-color: pink; color: blue}
h1 {font-size:30px}
p {font-size:15px}
</style>

Пример связанного стиля. Создать файл *.css, перечислить в нём селекторы и {свойства:определения}. В голове документа сделать связку с файлом. Атрибут media обозначает устройство вывода.

<link rel="stylesheet" type="text/css" href="*.css" media="all" />

Пример импортированного стиля. Используется в тексте CSS, а именно в головном разделе добавляется внутри элемента style, либо в файле CSS. @import url(*.css); Может использоваться для категоризации разных модулей базы стилей.

Комментарии /* */ работают только внутри CSS кода. Использование для известных целей. Можно использовать комментарии html-кода, но они не сработают на выполнение этих стилей.

Класс. Во второй главе выделять эту тему стало неуместно, поэтому обозначу здесь. Каждому конкретному элементу можно назначить класс, как например вот:

<b class="clasnm01"> </b>

Затем в разделе для css-кода, в списке селекторов можно обозначить .clasnm01 {свойство:значение} -- и все элементы, помеченные этим классом, примут стиль.

Для одного тега можно применить несколько классов. Приоритетность пока не понятна, но суть есть. Просто в атрибуте тега перечислить все классы через пробел. class="cl1 cl2 supercl". Класс переопределяющий в css будет ниже, верхний заменится.

В разделе для css-кода можно группировать селекторы, перечислив их через запятую, и наделив общим списком свойств. Пример: p, h1, a {color:red; font-size:18}.

Глава 2. Цвет, изображения.

Задать цвет можно несколькими способами. Для любого свойства, атрибутом которого является цвет, можно задать значение в виде ключевых слов, rgb(X,Y,Z)/(%,%,%), #ABCDEF. Сокращённый шестнадцатеричный используется, если значение имеет парные цифры - #AA11BB = #A1B. Для rgb() значение найти нетрудно, с помощью палитры онлайн-сервисов или в прикладных программах на пк.

Ключевые слова PINK ORANGE AQUA BLACK FUCHSIA GRAY GREEN LIME MAROON NAVY OLIVE PURPLE RED SILVER TEAL WHITE YELLOW. Теперь их намного больше, все печатать лень. Гугл под рукой.

Цвет фона. Применяется для любого элемента. Для элемента body задаёт фон всей страницы. Необходимое свойство с цветовым значением: {background-color: цвет}.

Изображение фона. Ставится на любой элемент. {background-image: url()} Изображение будет продолжено, повторяясь на всю страницу, если не определить свойствами такое поведение.

Заполнение также можно вести только горизонтально, вертикально, либо без заполнения. Вводится свойство {background-repeat:repeat-x}, значения соответственно repeat-x/repeat-y/no-repeat. Также есть значение repeat, но оно задаёт поведение, что и при отсутствии значения вовсе. Значение inherit пока непонятно, его рассмотрим позже.

Расположить изображение в определённом месте. {background-position: _ }, значения могут быть использованы в процентах, пикселах и ключевых словах. Два значения, первое сдвигает изображение вправо, по оси Х, второе вниз, по оси Y. Можно комбинировать значения в пикселах и процентах. Если задать одно значение, оно расположит изображение по горизонтали, а по вертикали по умолчанию встанет на 50%. Ключевые слова комбинировать можно только между собой. top right left bottom center. Разные варианты можно использовать, но из них получается только девять простых размещений по документу.

При прокрутке документа изображение может оставаться неподвижно относительно блоков документа, а может быть зафиксировано на экране. {background-attachment:scroll/fixed} Прокрутка изображения вместе с содержимим является стандартным поведением браузера, но можно переопределить значение. Применить можно к любым значениям, где в этом есть смысл.

Для свойства background-color можно задать значение transparent. {background-color:transparent} Если я правильно понял, это позволяет элементу стать прозрачным, и принять внешний фон. Вроде бы, это стандартное поведение всех нецветных элементов, поэтому переопределяется для отмены менее приоритетного стиля.

Сокращённые свойства. Доступны не для всех свойств. background - общий множитель, который разрешено вынести за скобку. Теперь два варианта, равнозначных по результату.

body {
background-color: white;
background-image: url(img01.jpg);
background-repeat: no-repeat;
background-position: center;
background-attachment:fixed
}

body {background: white url(img01.jpg) no-repeat center fixed}

background-size: [ <значение> | <проценты> | auto ]{1,2} | cover | contain

После объявления адреса изображения это свойство позволит масштабировать его.

Глава 3. Текст.

Есть пять категорий шрифтов.

    Вот их:
  • Serif -- с засечками, чем бы оно ни было.
  • Sans-serif -- без них.
  • Monospace -- равноширинные буквы.
  • Fantasy -- особо редкие.
  • Cursive -- рукописные, редкие.

body {font-family: arial, georgia, times, serif;}

Для тела будет использован первый из найденных перечисленных шрифтов. Унаследуется в другие элементы. Если ни один шрифт не будет найден, ключевое слово категории применит установленный в системе стандартный шрифт из категории.

Ключевые слова для размера текста. xx-small x-small small medium large x-large xx-large

p {font-size: medium}

Также размер можно задавать числовым значением для этого свойства. Измерениями являются em/ex/px - относительные, и pt/pc/in/mm/cm - абсолютные (относительно разрешения экрана, вероятно). Значение можно указывать в процентах, но оно будет всегда относительно. К примеру, задать для body свойство размера, а для заголовка проценты - они будут переопределены относительно этого значения.

p {font-size: 18px;}

Жирность даётся через свойство {font-weight:100-900}, 100 (lighter) - самый тонкий, 900 (bolder) - самый насыщеный, normal (400) - обычный, bold (700) - жирность. В скобках - ключевые слова.

Стиль даётся через {font-style: normal/oblique/italic}, второй стиль должен обозначить наклонный шрифт, либо поставить простой курсив, третий стиль должен просто дать курсивный текст.

Свойство {color: } даст цвет текста.

Свойство {text-align: left/center/right/justify;} соответственно в контейнере для абзацев разместит текст ровно вдоль левой стороны, центра, правой стороны, обеих сторон. Последний вариант всегда казался миловидным, поэтому здесь будет непременно.

Свойство {text-decoration: } имеет следующие значения. none - очистка, может убрать подчёркивание с ссылок. underline подчёркивает. overline "надчёркивает". line-through зачёркивает. blink - взмигивает. Тестировать не обязательно все обычные действия, а последнее заинтересовало.

Тестирование мигалки. Не работает у меня. Есть аналог html, но лениво использовать. <blink>Мигание</blink>

Свойство {text-indent: Xpx/%/-X} - установит отступ первой строки, для абзаца, к примеру.

Свойство {font-variant: } - имеет следующие значения. small-caps - делает капитель, т.е. все строчные символы немного увеличивает до почти заглавных. normal - по умолчанию. inherit - наследовать от родительского.

Свойство {text-transform: capitalize/lowercase/uppercase/none/inherit} - по первым трём пояснение, первый символ каждого слова устанавливается заглавным, либо все символы изменяются на прописные, либо регистр становится верхним. Без изменения либо наследовать - это понятно.

Свойство {line-height: } задаёт междустрочный интервал. {letter-spacing: } задаёт интервал между символами. {word-spacing: } между словами.

Уточнённый селектор - первая буква или первая строка текста.

p:first-line {color:red}
p:first-word {color:blue}

Сокращённый вариант включает свойства шрифта. Текстовые не включает. Необходим порядок значений.

p {font-style: italic;
font-variant: small-caps;
font-weight: bold;
font-size: 15px;
line-height: 20px;
font-family: arial, sans-serif;}

p {font: italic small-caps bold 15px/20px arial, sans-serif;}

Пока непонятно, как вытащить отсюда размер шрифта, но оставить межстрочный. Как отреагирует стиль на числовое значение жирности. И критично ли поменять положением ключевые слова, ведь они не пересекаются для других свойств.

{font-size-adjust; font-stretch; text-shadow}. Уравнивание размеров разных шрифтов, растягиваие текста, тень.

Глава 4. Ссылки, списки, навигация.

    Состояния ссылок.
  • a:link {свойства до активации}
  • a:visited {посещённая}
  • a:hover {под указателем}
  • a:active {в состоянии удержания нажатия}

Для примера сделаю отдельный класс, таким способом:

a.slnk01 {общие свойства, наследование, удобство}
a.slnk01:link {}

    Ссылки, в виде списка, увеличен размер, цвет, убрано подчёркивание:
  • Первый конспект.
  • Ссылка (нерабочая).
  • Ссылка (нерабочая).

Как вариант, для подуказательной ссылки можно применить изображение, чтобы выделить её особенно. Свойство {background-image: url();} уже известно.

Несмотря на чётко написанные псевдоселекторы, рекомендуется соблюдать порядок. Даже феньку приписали: LVHA (love-hate). Разумеется, смысл тут искать не нужно.

ID-селекторы. В коде стилей указывается список селекторов и свойств, для одного ID, затем в тексте только одному элементу/разделу можно назначить этот ID. Отличие от классов ощущается не сразу, но классом можно обозначить любое количество элементов.

<style>
#id01 {}
#id01 a {}
#id01 a:link {}
#id01 a:visited {}
#id01 a:hover {}
#id01 a:active {}
</style>

<div id="id01"></div>

Стиль для упорядоченных списков, свойство {list-style-type: } со значениями leading-zero (отсчёт с нуля), lower-roman (римские нижним регистром) и upper-roman (верхним). Заменить цифры изображениями - для каждого элемента списка задаётся свойство {list-style-image: url();}. В книге рекомендовалось каждый элемент обозначить отдельным классом, а в стилях каждый класс определить этим свойством с соответствующими значениями.

Стилизация неупорядоченных списков. Свойство {list-style-type: } со значениями disc (круг), circle (кольцо) и square (параллелепипед). Также значением none можно убрать маркеры. Заменить маркеры изображениями можно уже известным свойством, назначить его просто для элемента ul, элементы строк унаследуют маркер.

Свойство {list-style-position: } со значениями outside (маркер за пределом блока) и inside (в пределах) определит позицию маркера, и разницу в отступе строк.

{list-style: url() outside} -- вариант сокращения.

Для элементов можно использовать свойство {display: } со значениями inline - межтекстовый, и block - переводящий строку. Некоторые элементы резервируют блок документа в виде прямоугольника на всю ширину страницы, некоторые встраивают внутри текста. Для каждого элемента имеется своё значение по умолчанию. Это тема следующей главы, но автор решил уместным намекнуть при стилизации списка и ссылок.

Книга утверждает, что использовать список ссылок для навигации - очень мудро. Хотя пока технически отличий от простого наброска ссылок не наблюдается.

Глава 5. Поля, границы, отступы.

Вся структура документа представлена блоками. Элементы резервируют блоки встроенные или на всю ширину страницы. Поэтому при организации сайта очень удобно представлять структуру такими системами, блочную модель. Легче понимать и управлять отступами, границами, другими свойствами для каждого блока. Поскольку тут это уместно, стоит добавить о свойстве display {inline/block}, позволяющем контроллировать перенос строки в моменте блочного представления элементов.

Поле - расстояние от внешней границы к содержимому. Для элемента body это расстояние между общим контейнером и краем страницы. Даёт свойство {margin:}, сокращённо так обозначено.

Для класса ex5_01, применённого к абзацу, заданы свойства {border: 3px solid blue; margin-top: 50px; margin-left:10%; margin-right:30%; margin-bottom:0; background-color: white;}. Теперь понаблюдаем за реакцией. Отступ поля от верхнего блока - 30px. От левой стороны страницы - 10%. От правой стороны страницы - 30%. Граница 3px синего цвета - помогает продемонстрировать результат. Применять процент к вертикальным значениям поля не рискую. Вниз поле отступит на минимум, ноль. Цвет фона применился только к содержимому, поле не окрасилось.

Для абзаца с классом ex5_02 задано следующее: {border: 1px solid orange; margin-top:0; margin-left:0; margin-right:30%; margin-bottom:50px; background-color: green;}. Видно, что отступ от левого края остался, потому что элемент body имеет свой контур, и он является контейнером внутри страницы. К этому контуру и прижат абзац. Чтобы прижать к самому краю, нужно для тела назначить нулевое поле. Такое понятие, как "отступ самой страницы", технически отстутствует или пока неизвестно. Сверху поле тоже убрано, чтобы довершить слияние с предыдущим блоком.

Для блоков можно указывать отрицательное значение поля. Это сложно объяснить однозначно. Для горизонтальных моментов содержимое уходит за границу на расстояние модуля значения. Для вертикальных - приближает соседний блок на это расстояние, образуя пересечение.

Сокращение можно использовать по принципу trouble - верх право низ лево. {margin: 10px 20% 10px 20%}. Если указано два значения - поля будут симметричны попарно. Если одно - применится ко всем сторонам.

Границы даются свойством {border-TRBL-width/style/color:}. Немного грубое представление, но коротко и понять можно. Сначала граница, затем сторона, затем начертание. Обычно не приходится использовать разные стили для сторон одной границы, но попробовать надо будет.

Толщина может быть задана ключом - thin/medium/thick, либо размером.

dotted dashed solid double groove ridge inset outset hidden none

Наверное, нужны комментарии. Точки, штрихи, линия, две линии. Углубление, выступ, врезанный вид, рельеф. Два последних отличаются. Скрытую границу можно будет выявить скриптами, последняя вовсе не появится. (Класс ex5_03 для этого абзаца.)

Сокращения. Если нужна цельная граница, можно всё сгруппировать для {border:}, если нужно задать стороны - надо будет сделать четыре свойства {border-right: ; ...}.

Отступы. Это расстояние вглубь от границы, к содержимому. {margin: 0;}

Для класса ex5_04 использованы поля, но он призван показать отступы, поэтому их здесь тоже есть.
{
margin: 10px 20%;
border: 2px solid fuchsia;
padding: 15px 10%;
background-image: url(exp/phon1.jpg);
color:black;
}
Вот такой код. Поля вертикально 10 точек, по бокам - 20 процентов. Граница есть. Отступ в сокращённой форме, вертикально 15 точек, боковые - 10 процентов. Фон - картинка, цвет текста контрастно почернел. Так содержимое кажется на порядок миловиднее, а в этом есть смысл.

Глава 6. Позиционирование, плавающее размещение, z-индекс.

Позиционирование может объясняться как расположение относительно нормального потока структуры документа.

    Вот их: {position: }
  • absolute - выпадает из потока и устанавливается координатами в определённое место.
  • relative - не выпадает, но по желанию сдвигается на указанные расстояния.
  • fixed - размещается на окне просмотра и фиксируется там даже при прокрутке.
  • static - стандарт, переопределяет на обычное поведение браузера.

Плавающее. Оставаясь в потоке, размещается в определённой стороне, позволяя иным элементам - к примеру, тексту - обтекать плавающий элемент. Для элемента указывается свойство {float: } и сторона, где он будет размещаться. В нормальном потоке картинка синхронизировалась бы с первой строчкой текста, изменив её высоту и уменьшив благой вид. Хотя при использовании абзацев (блочных - не встроенных - элементов) это неактуально.

<img src="exp/gif01.gif" style="float:left; margin: 10px">

Глава 8. Анимация.

Нифига не понял. По css3 документации тьма, и тут эту анимацию надо несколько часов шерудить, чтобы вникнуть. А у меня как-то нет желания. Впрочем это всё равно надо будет пройти и кратко конспекнуть. Для любого тега почти можно задать animation-name. Это свойство сочетается с правилом ключевых кадров, названным по тому свойству. И вот эти кадры описывают поведение блока. Кроме имени, для анимации указываются ещё несколько свойств рядом, задержка повторы и прочее.
