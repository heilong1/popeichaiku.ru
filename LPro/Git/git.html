Курс по гитхабу. Краткое пособие по командной строке, то бишь ключевой функционал.


Глава 0. Гитхаб сайт.

Создать репозиторий, инициализировать ридми и гинигнором. Гитигнор компилировать в сервисе.

Глава 1. В консоли

git --version данная команда отображает версию, проверка факта установки.

git init // создать локально с папкой .git

git status // сводка по ветке, комитам, файлам

git add name // фиксация файла

git add . // фиксация всего

файл .gitignore прописать столбиком файлы и папки, которые не будут фиксироваться. гитигнор тоже комитить

git commit -m "text" // осуществить запись всех фиксаций

ветки

git branch // список

git branch name // создать

git branch -d name // удалить

git chekout name // переключиться

git chekout -b name // создать и перекл

git merge name // впитать ветку

на гитхабе создать аккаунт и удалённый репозиторий

git config --global user.name // отобразить логин гитхаба

git config --global user.email // отобразить емел

git config --global user.name "name" // менять логин, и емейл так же

git remote add origin eadress.git // связать по адресу репозита

git push -u origin master // первая отгрузка данных на репозит

git push // отрузка на репозит

git clone eadress.git // загрузить репозит по адресу

git pull // откачать изменения с сервера, возможны зеркальные локи

Глава 2. В декстоп прог

Возобновление работы не исследовано. Мой прогресс сохранён от старта использования.

Сначала лучше создать репозиторий на сайте, и инициализировать. Затем в приложении клонировать его в пустую папку, и докачать туда нужные файлы.

После изменения содержимого папки, в нижнем левом углу программы можно собрать изменения в коммит. Храниться он будет локально. В верхней строке определится кнопка Push origin, которая пересылает коммиты в гитхаб.

Ветки. Создавать, пулреквестить на сайте, фетчить мастера в приложении.
